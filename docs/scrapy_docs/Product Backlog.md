

---

Let's define the different pre-requisites : 


- The users should be able to authenticate(Sign In/Sign Up) on the platform on a role-authentication basis 
- The users of the platform are: farmers, decision-makers and system admin. 



--- 


# User Stories 


1. ~~As a farmer i should be able to authenticate in order to use use the platform~~ 
2. As a farmer i should be able to insert the geolocation of my fields in order to geo tagged them in the platform 
3. As a farmer i should be able to log the area of cultivation in order to delimit my cultivation area on the interactive map 
4. As a farmer i should be able to log in crops variety, yields estimation in order to provide data about my farms 
5. As a farmer i should be able to report a pests/diseases outbreak in order to report outbreak
6. ~~As a farmer i should be able to get notified about outbreaks in near real-time in order for me to take mitigation measures~~
7. ~~As a decision-maker, i should be able to authenticate in order to use the platform~~ 
8. As a decision-maker, i should be able to visualize in real-time on an interactive map the evolution of outbreaks in order to plan a large-scale mitigation initiative
9. As a decision-maker, i should be able to modify the level of threat of outbreak report in order to ease the outbreak prioritization process 
10. ~~As a decision-maker, i should be able to notify a particular set of farmers in order give them mitigation directives~~ 
11. As a system admin, i should be able to create, read, update and delete decision-makers account in order to manage the access to the decision-makers interface
12. As a system admin, i should be able to monitor the logging activities of each decision-makers account and the actions they performed while logged in to provide accountability tracing in the platform
13. ~~As a system admin i should be able to authenticate on the platform in order to administrate the system~~
14. ~~As a dm , i should be able to get notified about sysadmin modif on my account~~
